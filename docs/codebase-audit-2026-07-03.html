02P0 — critical (all fixed)

Data-integrity, security-boundary and pilot-visible faults. These were the priority.

P0·1
Skill database silently lost all multi-part maths data
Duplicate skill dimensions across parts tripped a Postgres upsert error that was swallowed, dropping the whole submission's rollup — starving the product's strategic asset. Now aggregates per-dimension before write, with a dedupe safety-net in recordSkillSignals.
Fixed
P0·2
Stored XSS: student maths → teacher's session
The shared escaper didn't escape quotes; student MathLive strings landed inside HTML attributes on the marking page. Made the escaper quote-safe across 17 pages — closes the whole attribute-injection class.
Fixed
P0·3
Any student could self-promote to teacher
set-role had no transition guard, so every "teacher-only" gate was decorative. Now refuses student→teacher self-promotion; teacher status must come from LTI or an admin grant.
Fixed
P0·4
Faculty-restricted leaders saw whole-school data
Multi-faculty leaders got an unfiltered card corpus and shared a cache row with unrestricted executives. Now filters the corpus to granted KLAs and folds the restriction into the cache key.
Fixed
P0·5
Every AI draft posted 100% "FullyGraded" to Canvas
Parent- and student-visible during the PCS pilot. Drafts now post Submitted / Pending with no score; real grades still post correctly; grade-clears now pass back; /scores URL is query-string safe.
Fixed
P0·6
One bad roster member aborted the whole NRPS sync
A single pre-existing account left the rest of the class un-enrolled, silently. Now per-member resilient with a logged skip summary; launch.ts given a 300s timeout.
Fixed

03P1 — correctness (fixed)

Bugs that produce wrong behaviour or wrong numbers under realistic conditions.

P1·2
Fail-open lock & cap sweep
Locks and the 3-draft cap silently vanished on multi-row edge cases or DB hiccups. Now use .limit(1) and fail closed; cap reads return 500 before spending Sonnet.
Fixed
P1·1
Silent 1000-row truncation on insights queries
At scale every card computed over an arbitrary, unordered subset (wrong stats + flapping cache fingerprint). Added deterministic pagination.
Fixed
P1·3

Any user with a task UUID could freeze its content by creating an empty autosave. Now requires membership + published task + non-empty content.
Fixed
P1·6

Moderation edits that netted to zero left stale decile cards "fresh." Fingerprint now hashes per-row id+mark tuples (v3).
Fixed
P1·7·8

Fixed
P1·5

A partial re-grade deleted every other question's mark and deflated the LMS total. Now merges; omitted fields keep their stored values.
Fixed
P1·4·9·11

Maths now 400s on stale part-ids instead of charging a draft for empty feedback; attachment downloads bind the path to its owner; profiles flip stale on delete and regen on any count divergence.

06The insights layer — the big opportunity

Structural finding

Nothing in the insights layer reads the skill database.

student_skill_profile — the per-student, per-dimension data you've called the moat — is read by exactly one feature (the Lesson Differentiator). Every insight staff see is either raw-mark arithmetic or an LLM re-deriving patterns from feedback prose: expensive, rate-limited, unquantified, and unstable between runs. You're paying Sonnet to approximate numbers you already hold.

The correctness fixes above were the prerequisite. These build the actual product on top of data that's now complete and correctly scoped.

R1 · highest leverage START HERE
Cohort skill matrix
A class × dimension heatmap straight from the skill DB — "14 of 22 students at developing or below on W4 Use of evidence, trend flat." Free, instant, quantified, identical every load. Turns the taxonomy from invisible plumbing into the visible, defensible product.
R2
Skill observation history
A thin history table (backfillable) enables growth-over-time — the mark-free developmental report schools actually want, and the retention story: data compounds every term.
R3
Feed the skill profile into every LLM synthesis
Anchor narratives in structured levels/trends alongside the prose — cuts run-to-run variance and shrinks the prompt.
R4
Rebuild improvement velocity on skill deltas
Replaces the noisy verbatim-string matching with per-dimension deltas between drafts. Same card, real numbers, no LLM call.
R5
Close the insight → action loop
Every gap card shows counts + affected students, with one click to spin up a Lesson Differentiator task targeting that dimension. Makes insights start the flywheel instead of ending in reading.
R6
Provenance on every card
"Based on 43 submissions across 6 tasks, past 12 months, 3 below confidence threshold." An experienced teacher hedges on thin evidence; the cards should too — cheapest trust-builder with sceptical staff.
